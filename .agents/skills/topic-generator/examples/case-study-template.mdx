## Read & Write Paths

### Write Path (Creation)
1. Client sends request to API Gateway
2. API Gateway authenticates and rate-limits
3. Service validates input and generates unique identifier
4. Data is written to primary database
5. Cache is updated/invalidated
6. Response returned to client

### Read Path (Retrieval)
1. Client sends request to API Gateway
2. Check cache first (cache-aside pattern)
3. If cache miss → query database
4. If found → populate cache, return response
5. If not found → return 404

## Scaling Strategy

### Stage 1: 0 → 10K Users
- Single server, single database
- Local caching (in-memory)

### Stage 2: 10K → 1M Users
- Separate DB from application server
- Add Redis cache layer
- Introduce load balancer + 2-3 app servers
- Read replicas for DB

### Stage 3: 1M → 100M Users
- Database sharding
- CDN for static assets
- Message queue for async processing
- Microservices extraction for hot paths

### Stage 4: 100M → 1B Users
- Multi-region deployment
- Global load balancing
- Dedicated analytics pipeline
- Feature flags for gradual rollouts

## Failure Scenarios & Production Considerations

<!-- 🔴 ADVANCED LAYER — what breaks in production and how real systems handle it -->

### What Can Go Wrong?

| Failure | Impact | Mitigation |
|---|---|---|
| Cache layer goes down | All requests hit DB → potential overload | Circuit breaker on cache, fail open, DB can handle 10x burst for 5 min |
| Database primary fails | Writes unavailable | Auto-failover to replica (< 30s), promote replica to primary |
| Network partition | Split-brain risk in distributed components | Quorum-based writes, conflict resolution strategy |
| Sudden traffic spike (10x) | Service degradation, timeouts | Auto-scaling, rate limiting, load shedding, circuit breakers |

### Production Monitoring
- **Key metrics to watch**: p99 latency, error rate, cache hit rate, DB connection pool usage, queue depth
- **Alert thresholds**: Error rate > 1% for 2 min, p99 > 500ms for 5 min, cache hit rate < 70%
- **Runbooks**: Document the top 5 incident types and their resolution steps
